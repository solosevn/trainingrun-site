TRScode Methodology

Last Updated: February 19, 2026

TRScode is the daily AI coding leaderboard measuring who writes the best code — across 8 independent sources and 5 pillars. Unlike TRSbench (which measures overall capability), TRScode measures pure coding ability. This page documents our complete methodology with full source citations.

Why TRScode Exists

AI models are increasingly used to write, debug, and review code. But which models actually write the best code? Existing benchmarks test individual tasks — not comprehensive coding ability across multiple dimensions. TRScode fills this gap with 5 independent pillars drawn from 7 coding benchmark platforms. Every score is derived from real coding tasks with verifiable outcomes. No synthetic benchmarks. No self-reported metrics. Just real coding performance.

The 8 Sources

SWE-bench Verified
17% (Pillar 1)

lmarena.ai

The Formula

TRScode = SWE-bench Verified (17%)
+ SWE-rebench (13%)
+ LiveCodeBench (15%)
+ BigCodeBench (10%)
+ Terminal-Bench Hard (12%)
+ SWE-bench Pro (8%)
+ SciCode (15%)
+ Arena Code Elo (10%)

Scoring Method: Models with null sub-metrics receive zero (Option B scoring). All weights sum to 1.00.

Why These 8 Sources

  • 7 independent organizations — no single org controls the score
  • 4 of 7 use continuous refresh or decontamination
  • Multi-dimensional: bug fixing + code generation + terminal agents + scientific coding + human preference
  • 100+ unique models across all sources combined
  • All sources are free, public, and independently verifiable

Data Integrity

Every daily update includes an SHA-256 checksum. The chart page verifies this checksum on load and displays a warning if data has been modified outside the official pipeline.
